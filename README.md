# Quant Risk: EVT + Опционы для управления риском портфеля

[![Python](https://img.shields.io/badge/Python-3.14+-blue.svg)](https://python.org)

> Реализация ключевых методов управления финансовыми рисками **своими руками** без использования готовых высокоуровневых библиотек.

## О проекте

Проект демонстрирует практическое применение методов количественного анализа рисков, используемых в банках и хедж-фондах. Основной фокус — **моделирование хвостов распределения** (Extreme Value Theory) и **оценка рыночного риска** (VaR, Expected Shortfall).

### Что реализовано "руками"

| Модуль | Реализация | Библиотеки |
|--------|------------|------------|
| **Генерация данных** | t-распределение, Markov switching волатильности | NumPy |
| **EVT / GPD** | Метод превышений порога (POT), MLE для GPD | SciPy (minimize) |
| **Опционы Блэка-Шоулза** | Ценообразование, греки (Delta, Gamma, Vega, Theta) | pure NumPy |
| **Бэктестинг VaR** | Тест Купика, тест на независимость, loss-функция | pure NumPy |
| **Риск портфеля** | VaR/ES, стресс-тестирование, корреляционная матрица | NumPy, Pandas |

### Что проект умеет

- Генерировать реалистичные финансовые данные с кризисными режимами
- Моделировать **толстые хвосты** с помощью GPD (Generalized Pareto Distribution)
- Сравнивать **VaR и ES** для нормального, исторического и EVT подходов
- Оценивать **опционы** и их чувствительность к рыночным факторам
- Тестировать качество VaR-моделей через **тест Купика**
- Проводить **стресс-тестирование** портфеля

### Что я узнал
    Extreme Value Theory (EVT)
    Почему EVT важна : нормальное распределение недооценивает вероятность экстремальных убытков (кризисы 2008, 2020)

    Теорема Пикандса-Балкема-де Хаана: распределение превышений над высоким порогом сходится к GPD

    Параметр ξ: ξ > 0 — тяжелый хвост (Фреше), ξ = 0 — экспоненциальный, ξ < 0 — короткий

    VaR vs Expected Shortfall
    VaR говорит: "с вероятностью 95% убыток не превысит X"

    ES говорит: "в худших 5% случаев средний убыток составит Y"

    Базель III рекомендует ES вместо VaR, так как ES субаддитивен

    Опционы и греки
    Дельта — скорость изменения цены опциона при изменении базового актива

    Гамма — скорость изменения дельты (важно для хеджирования)

    Вега — чувствительность к волатильности
